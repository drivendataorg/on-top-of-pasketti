# Solution - On Top of Pasketti: Children's Speech Recognition Challenge

## Summary

The solution fine-tunes [Qwen3-ASR-1.7B](https://huggingface.co/Qwen/Qwen3-ASR-1.7B), an encoder-decoder speech-language model, on the competition training data (~342k utterances) supplemented with ~34k TTS-generated single-word utterances. Training uses KL distillation against the frozen pretrained model (alpha=0.5, cosine annealed to 0) to regularize fine-tuning and prevent catastrophic forgetting. Duration-bucketed batching groups utterances of similar length to minimize padding waste.

At inference time, the full `Qwen3ASRThinkerForConditionalGeneration` model runs batched greedy decoding with duration-adaptive `max_new_tokens` limits to avoid wasted computation on short utterances. Audio is loaded as 16kHz mono via librosa. No beam search, no ensembling. VAD is bundled but disabled by default; it only serves as a fallback chunker for audio exceeding 30 seconds.

Key techniques:
- **KL distillation**: frozen copy of the pretrained model provides a soft-target regularizer during training, preventing the model from overfitting to the (noisy) children's speech transcripts
- **TTS data augmentation**: ~34k synthetic single-word utterances generated with Qwen3-TTS voice cloning, targeting the short-utterance vocabulary that dominates the competition data (236 SLP diagnostic words x 144 child speakers)
- **Duration-bucketed batching**: both training and inference sort utterances by duration to minimize padding and maximize throughput

```bash

micromamba run -n child_asr pip install -r requirements.txt
```

# End-to-end

To reproduce the full solution from scratch (data extraction, TTS generation, training, packaging, and inference):
```bash
bash run_solution.sh
```

**Prerequisites**: The competition data must be placed under `data/` before running:
```
data/
├── drivendata_data/
│   ├── audio_part_{0,1,2}.zip          # DD speech audio (3 parts, ~12 GB)
│   ├── noise_part_{0,1}.zip            # Noise audio (2 parts, ~5 GB)
│   ├── train_word_transcripts_dd.jsonl # DD training transcripts (95k)
│   ├── val_word_smoketest.jsonl        # Validation set (9k, held-out smoketest IDs)
│   └── submission_format_aqPHQ8m.jsonl # Test set A
└── talkbank_data/
    ├── talkbank_audio.zip              # TalkBank speech audio (~21 GB)
    └── train_word_transcripts_tb.jsonl # TB training transcripts (255k)
```

This runs the following steps:
1. **Step 0**: Extract audio zips, merge transcripts, create train/val split, set up 9k smoketest evaluation directory
2. **Step 1**: Generate ~34k TTS augmentation utterances (~2 hours)
3. **Step 2**: Fine-tune Qwen3-ASR-1.7B with KL distillation (~17 hours)
4. **Step 3**: Find best checkpoint by validation WER
5. **Step 4**: Package submission (extract weights to safetensors)
6. **Step 5**: Run inference via Docker against 9k smoketest

Each step is described in detail below.

# Run training

## Step 1: Generate TTS augmentation data

236 SLP diagnostic keywords are synthesized via Qwen3-TTS voice cloning using 144 child speakers sampled from the training data (evenly across age buckets). This produces ~34k single-word utterances.

```bash
export PYTHONPATH=.
micromamba run -n child_asr python src/data/generate_tts_words.py
```

Output: `data/tts_words/tts_words_manifest.jsonl` and `data/tts_words/audio/*.wav`

## Step 2: Fine-tune Qwen3-ASR-1.7B

Training is driven by YAML config files. The winning run used `config/training/word/qwen_distill_augment.yaml` with the following overrides:

```bash
export PYTHONPATH=.
micromamba run -n child_asr python src/models/word/finetune.py \
    --config config/training/word/qwen_distill_augment.yaml \
    distillation.kl_alpha=0.5 \
    training.epochs=5 \
    training.validate_first=false
```

The full training config:

```yaml
model:
  type: qwen
  name: Qwen/Qwen3-ASR-1.7B
  freeze_audio_encoder: false
  freeze_lm_layers: 0

data:
  extra_manifest: tts_words/tts_words_manifest.jsonl

distillation:
  kl_alpha: 0.5
  kl_alpha_min: 0.0

training:
  epochs: 5
  batch_size: 16
  max_batch_duration: 60
  sampler: duration_bucket
  gradient_accumulations: 4
  precision: bf16-mixed
  gradient_clip_val: 1.0
  seed: 42
  num_workers: 4
  val_check_interval: 0.5

optimizer:
  lr: 1.0e-05
  weight_decay: 0.01
  warmup: 0.05

checkpoint:
  save_top_k: 2
  monitor: val_wer_norm
  mode: min
```

Checkpoints are saved to `models/qwen_word/`. The best checkpoint is selected by lowest `val_wer_norm`. The winning run achieved val_wer_norm=0.1731. The full training log is at `logs/finetune/20260316_143800.log`.

Model weights total ~3.8 GB (safetensors format). Weights will be made available as part of the competition's solution release.

Data sources:
- Competition training data: `data/drivendata_data/train_word_transcripts_dd.jsonl` (95k) and `data/talkbank_data/train_word_transcripts_tb.jsonl` (255k), merged by Step 0 into `data/raw/train_word_transcripts_nosmoketest.jsonl` (341k train) with smoketest IDs held out as validation (9k)
- TTS augmentation: `data/tts_words/tts_words_manifest.jsonl` (34k), generated in Step 1
- Noise audio: `data/raw/noise/` (extracted from noise zips in Step 0, available for augmentation experiments)

# Run inference

## Standalone local inference (without Docker)

To generate predictions using the pre-trained model weights directly on the host machine:

```bash
bash run_inference.sh <MODEL_DIR> <AUDIO_DIR> <INPUT_JSONL> [OUTPUT_JSONL]
```

Arguments:
- `MODEL_DIR` — path to the Qwen3-ASR model directory containing `model.safetensors` and tokenizer files (default: `submission/qwen_model`)
- `AUDIO_DIR` — directory containing `.flac` audio files named by utterance ID (default: `data/raw/audio`)
- `INPUT_JSONL` — JSONL file with `utterance_id` fields specifying which utterances to transcribe (default: `data/raw/submission_format_aqPHQ8m.jsonl`)
- `OUTPUT_JSONL` — output path for predictions (default: `predictions.jsonl`)

Examples:
```bash
# Run on competition test set
bash run_inference.sh submission/qwen_model data/raw/audio data/raw/submission_format_aqPHQ8m.jsonl predictions.jsonl

# Evaluate on 9k smoketest validation set
bash run_inference.sh submission/qwen_model runtime/data-smoke/word/audio runtime/data-smoke/word/submission_format.jsonl smoketest_preds.jsonl
```

The script loads the model, sorts utterances by duration for efficient batching, runs batched greedy decoding, and falls back to single-utterance decoding on OOM. Inference on 9k utterances takes ~2 minutes on GPU.

## Export model weights

To export fine-tuned weights from a PyTorch Lightning checkpoint to safetensors format (without the full submission packaging pipeline):

```bash
bash export_model.sh <path/to/checkpoint.ckpt> [output_dir]
```

This loads the pretrained Qwen3-ASR-1.7B base model, swaps in the fine-tuned weights from the checkpoint, and saves the complete model + tokenizer to the output directory (default: `exported_model/`). The exported model can be used directly with `run_inference.sh`:

```bash
bash export_model.sh models/qwen_word/best.ckpt exported_model
bash run_inference.sh exported_model data/raw/audio runtime/data-smoke/word/submission_format.jsonl
```

## Package the submission

The packaging step extracts fine-tuned weights from a PyTorch Lightning checkpoint and bundles them with the inference script:

```bash
bash package_submission.sh qwen <path/to/best_checkpoint.ckpt>
```

This creates a self-contained submission directory with the model in safetensors format, ready for Docker evaluation.

## Run inference (Docker)
### Run with Docker

Test the submission against the 9k smoketest using the competition Docker runtime:

```bash
cd runtime && KIDSASR_DATA_DIR="$(pwd)/data-smoke/word" just track=word run
```

This pulls the official competition Docker image and runs the submission inside it with network isolation, matching the competition environment exactly.

### Build and run with local GPU Docker

For local GPU testing (e.g., on Blackwell GPUs not supported by the official image):

```bash
# Build local image
cd runtime && just gpu-build

# Run with local GPU image
cd runtime && KIDSASR_DATA_DIR="$(pwd)/data-smoke/word" just track=word gpu-run
```

The submission inference script (`submission/main_qwen_word.py`):
1. Loads the fine-tuned Qwen3-ASR model from `qwen_model/` in safetensors format
2. Reads `submission_format.jsonl` to get utterance IDs
3. Loads each audio file as 16kHz mono via librosa
4. Groups utterances into duration-sorted batches (max 300s cumulative per batch)
5. Runs batched greedy decoding with duration-adaptive `max_new_tokens`
6. Falls back to single-utterance decoding on OOM
7. Writes predictions to `submission.jsonl`

No preprocessing, external data, or network access is required at inference time. The only input is the raw competition audio files and the bundled model weights.
